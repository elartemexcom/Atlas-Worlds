Proposal: A Spatial Intelligence System for Visual Archives

Atlas is a Geometry of Meaning that turns images into places, intersections, and worlds.

Traditional organization is linear: folders, filenames, search queries. Atlas is relational and spatial: it lets meaning behave like physics. Users place Tags and Anchor Images, tune Heat, and watch a collection settle into navigable Clusters and ranked Columns.

Geometry of Meaning

The core idea: meaning is not a list. It is a shape. Atlas gives meaning a coordinate system, a gravitational logic, and visible formations. This single lens unifies the entire roadmap: Tag Fields, Spheres, Dimensions, D10 story dice, and ultimately world traversal.

The problem

Why existing tools break for visual minds

Folders demand upfront categorization. Search demands you remember words. Galleries show items, but hide relationships. Tags exist, but remain static labels instead of forces that reshape discovery.

Rigid

Folders are pre-commitment

You must decide what something “is” before you’ve explored what it relates to.

Memory-dependent

Search assumes recall

If you forget the name, you lose the thing. Atlas assumes recognition instead.

No gradients

Relationships are hidden

Most systems cannot show “closest to this vibe, then slightly further, then strange cousins.”

The solution

Atlas replaces lists with formations

Atlas converts images into embeddings, then makes exploration spatial. Users don’t ask “where is that file?” They ask “what’s closest to this meaning?”

1) Ingest: images → embeddings + metadata
2) Place: tags + anchors → coordinates/forces
3) Weight: heat (default neutral equilibrium)
4) Form: columns, clusters, crossroads
5) Browse: global view + local neighborhood + depth dials
6) Learn: user curation/annotations improve models
  • Default heat shows the most faithful map of what the archive naturally contains.
  • Increasing heat intentionally biases mass toward a chosen meaning and away from others.
  • Tag placement controls which intersections become sharp (close) vs broad (far).
  • Anchors add aesthetic gravity: “like this image, but in the direction of these tags.”
Explainability

Every result is readable

Atlas isn’t a black box. Every image and formation carries a compact explanation bundle so users can learn the map.

Each image stores:
Field Coordinate Column Coordinate Heat Context Tag Relations Anchor Relations

Each column stores a Column Signature: a fingerprint of tag positions, tag heats, active anchors, and the column’s identity. This enables side-by-side comparison and “return to this exact formation” navigation.

Atlas versions

v1, v2, v3 are stepping stones, not rebrands

Each version keeps the same essence (meaning as geometry) but expands how intersections form and how humans navigate them.

Atlas v1.0

Spatial Tag Field → Columns

A flat XY Tag Field where tags are placed like stars. Under intersections, ranked columns form. Best matches at the top, variation down the column.

  • Drag tags closer for tighter intersections, further for broader blends
  • Heat adjusts how strongly a tag pulls images into its basin
  • Anchor images optionally bias the field
Atlas Sphere v2.0

Bounded world → Interior placement

The archive becomes a finite world. Tags and anchors sit on the surface. Every image is placed inside the sphere where it best approximates surface forces.

  • Prevents infinite sprawl: clusters become revisitable regions
  • Click a point to open a local neighborhood gallery around it
  • Bookmark and annotate regions: you become a Cartographer of Meaning
Atlas Dimensions v3.0

Intersections become multi-dimensional formations

v3 adds a crucial leap: tags are not only forces, they are also placed in an XY micro-field that determines how multi-column formations appear beneath and between them.

  • Single image + tag can form a branch (a directed column)
  • Two images can form a pairing branch: “best approximates both” ranked top→down
  • Multiple tags produce multi-column stalactite-like formations under tag intersections
  • Moving tags closer/further changes which intersections become “power nodes”

Heat, clarified

What “heat pulls mass” means in plain terms

With equal heat, Atlas tries to place images where they “most truthfully” belong relative to the tag geometry. Increasing a tag’s heat increases its influence in the scoring function, so more images qualify as “best fits” for that tag. This can pull images away from neighboring tags because the system is optimizing under weighted forces.

Default heat

Neutral equilibrium

All tags contribute evenly. The map reflects the archive’s natural structure, not a user’s bias.

Raising heat

Intentional bias

A hot tag expands its basin. More images get pulled toward it, even borderline matches, reshaping clusters and columns.

Lowering heat

Softened influence

A cooled tag becomes a gentle hint rather than a command. Intersections become broader and less exclusive.

Training pipeline

D10 Book-Dice: turning books into spatial meaning engines

The D10 model converts long-form text into structured meaning columns. Each page becomes a tile on a 10-sided die (or a set of dice), and each tile generates: a page summary (relative to the whole work) plus a controlled set of tags. Those tags become constraints that Atlas uses to rank and settle images into columns and slices.

Input: Book (N pages)
Step A: Page → summary (page-in-context-of-whole)
Step B: Tags generated with anti-repeat constraints
Step C: Tags → column constraints → ranked image gradients
Step D: Human curation + model interpolation produce best columns
Output: 1M+ “correct” meaning dice used to train robust mapping from text→visual formations
Tag novelty rules

Novel, non-redundant, but still accurate

The goal is not random tags. The goal is coverage and distinctiveness. A page should be described in a way that is correct, but not a clone of prior pages.

  • Tag 1 is always unique: the best single encapsulation of that page’s uniqueness within the work.
  • Remaining tags prefer novelty; if reuse happens, order changes to reflect the page’s emphasis.
  • Constraint: avoid producing a tag block that is “half the same” across pages.
  • Operational target: ~40 tags/page (tunable by scale and cost).
Human loop

How we train meaning into visuals

Curators and artists generate two anchors per column: the closest interpretation and the farthest interpretation. Models generate intermediate images in a gradient. Users then mix and arrange columns to select the best formation. Those selections become training data.

  • Artist produces: Top Anchor (closest) + Bottom Anchor (farthest)
  • Model produces: 10–50 intermediates (a graded column)
  • Community selects and ranks the best column structure
  • Repeat at scale: target 1M+ columns to train robust text→visual mapping
AtlasWorld + Genesis

From formations to worlds: Atlas Genesis

AtlasWorld places images on a sphere and lets meaning settle within the constraints of that world. Atlas Genesis goes further: you can drop an image at a coordinate and the system transforms it toward what “belongs there,” using the original as a seed and drawing style/structure from learned worlds.

AtlasWorld

Images on a sphere

Users place images and tags on spherical coordinates. The archive arranges itself within that bounded world. Best-ranked worlds become training exemplars.

Genesis Engine

Coordinate-driven transformation

Drop an image at a coordinate and the system alters it toward what typically lives there, guided by millions of prior globes and user preferences.

Highlighting meaning

Most-chosen clusters become lessons

The system tracks which clusters are frequently visited, bookmarked, or loved. It can recommend coordinates with human-readable “reasons.”

Style by region

Different hemispheres, different aesthetics

Regions can carry preferred styles. A world can be published, populated by users, and the best-ranked worlds become training seeds. “Style geography” becomes a platform feature.

  • Coordinates can represent aesthetic rules (palette, texture, composition, tone)
  • Users can publish worlds as shareable artifacts
  • Top worlds feed training and recommendation systems
Profile-driven discovery

Bookmarks become preference signals

The system builds a taste profile from actions users already understand: save, love, bookmark, time spent. Then it can predict what you’ll likely love in new generations and clusters.

  • Non-invasive by default: preference from explicit actions
  • Optional advanced signals must be opt-in and clearly explained
  • Users can organize their own libraries with tags for stronger personalization
Investor demo layer

VR Slices: turning meaning into traversable worlds

A slice is a 2D plane taken from the sphere at chosen coordinates (x, y). No depth required. The slice contains a curated neighborhood of images that share a motif fingerprint. An AI interprets the slice as a coherent “place” and spreads landmarks across a walkable world.

Slice

40-neighbor neighborhood

Select a coordinate and open a neighborhood slice, then render it as a world of landmarks: buildings, objects, characters, motifs, and routes.

Coherence

System-level interpretation

The AI sees the slice holistically, then arranges related elements near each other: pump near pump-like building near pump-like vehicle.

Traversal

Walk, ride, drift

The user traverses by foot, vehicle, or themed motion. Meaning becomes geography, not a grid.

Portals

Portals are intersections, not teleportation

A portal exists where two slices overlap in motif space. The form of the portal is generated from shared motifs: fog gate, rabbit hole, mirror, tunnel, drain, door crack. Transitions become semantic continuity.

Portal = Intersection(Slice A, Slice B)
Form = Function(shared motifs)
Result: travel through meaning, not menus
Auto-naming

Slices get names humans remember

The system names slices using motif density and tone: searchable, memorable, and consistent. Example: creepy-cute-goth-gf-silent-hill-theme-haunted-disney.

  • Names act as compressed semantic fingerprints
  • Users can bookmark regions by name and coordinate
  • Guided tours can be created: routes, notes, reasons
Community engine

Creative nudges, not invasions

Atlas improves when people label well. The system can reward meaningful, varied tagging without requiring invasive tracking. The key is transparency and user benefit.

Nudge

“You’ve used that tag 500 times”

The UI can suggest modifiers or new phrasing. Users can continue, or add a variant that better captures nuance.

Celebrate

“You made a new tag”

Reward creativity with clear feedback: meaningful contributions are recognized, and users feel they are building a shared atlas.

Quality

“That’s not sensical” (gentle)

A quality check can flag nonsense tags and encourage better phrasing. The user stays in control. The goal is clarity, not punishment.

Rewards

Real benefits for meaningful tagging

Reward structure should be understandable and motivating: small-to-us, meaningful-to-them. Benefits can be credits, premium features, or early access to higher-quality generations.

  • Monthly goal: “Create 100 meaningful new tags” → unlock benefits
  • Human review can be explicit: users know quality is evaluated
  • Rewards align with user value: better organization, better discovery, better generations
Trust

Privacy and consent as a feature

Atlas can function fully with non-invasive signals (save, love, bookmark, time-in-view). Anything beyond that must be opt-in, explained, and beneficial to the user.

  • Default: no camera, no biometrics, no hidden tracking
  • Optional advanced feedback requires clear consent and clear payoff
  • Users can view, export, and delete their preference profile
