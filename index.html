Executive Summary

Readable in one pass

Atlas turns image archives into a navigable, similarity-driven space where users explore by relationship instead of file paths. It replaces static folders and grids with clusters, columns, and gravity fields.

Across three versions, Atlas evolves from a clean image-to-image map into a full dimensional organization system where tags live on an XY field, pull images with adjustable heat, and generate multi-column structures that can be explored by depth using layer scrolling and axis views.

Key Terms and Unique Features

Atlas vocabulary

Cluster of Meaning A navigable region of images that share a coherent visual or conceptual identity.

Gravity Well A force-like pull created by images and tags that draws related images toward a region or column.

XY Tag Field A 2D plane where tags can be positioned. Tag proximity reshapes how columns and intersections form.

Heat A weight on a tag that increases its pull. Higher heat absorbs more images into that tag’s influence.

Column A ranked vertical sequence of images, typically from closest to furthest relative to an anchor concept.

Stalactites A visual metaphor for multi-column formations that appear beneath tag intersections, like hanging icicles.

Depth Carousel A UI control that lets users scroll through layers (top to bottom) of a column or cluster.

Axis Scroll Viewing the cluster from different perspectives: Z-layer overview, plus side views for comparison.

Unique Complex Marker A compact identifier describing a column’s tag configuration, positions, and heats.

Coordinate Descriptor Metadata that explains an image’s relationship to all tags, not just nearest tags.

At-a-glance: Tags are placed in XY space → tags have heat → heat + proximity reshapes gravity → gravity forms columns and intersections → depth carousel lets you traverse from strongest matches to weakest matches → axis views let you compare columns and slices.

Atlas v1.0: Plain Image Atlas

Foundational version

What v1.0 does

Atlas v1.0 is a clean, modern replacement for folder browsing. Users pick an image as a hub, and Atlas returns the closest images by similarity. This creates an immediate, intuitive sense of “what belongs near what”.

Input: Images Output: Similarity neighbors UI: Hub-based exploration Core value: Fast discovery

Unique v1.0 features

  • Hub Selection: click an image to make it the center of gravity
  • Similarity Field: nearby images appear closer; distant ones farther
  • Cluster of Meaning: users can “walk” through similarity neighborhoods
Investor-simple explanation: v1.0 replaces “where is the file?” with “what is closest to this idea?”

Atlas v2.0: Atlas Sphere

World navigation layer

What v2.0 adds

Atlas v2.0 introduces a sphere-based navigation metaphor for exploring large archives as a continuous world. Instead of a flat map, images can be explored through spherical “slices” around a chosen axis, creating an intuitive sense of direction, neighborhood, and distance.

New: World surface navigation New: Local slice view (around a coordinate) New: Coordinate + text pairing Goal: Explore by place, not filename

Key v2.0 mechanics

  • Sphere Coordinates: images can be associated to points on a globe-like surface
  • Slice View: click a point and display a ring or patch of nearby images (example: ~40 neighbors)
  • Scrolling Exploration: smoothly travel to nearby regions, rather than jumping lists
What v2.0 is for: making a huge archive feel like a place you can traverse and remember.

Atlas v3.0: Atlas Dimensions

Full dimensional gravity system

What v3.0 adds

Atlas v3.0 is where the system becomes a true spatial intelligence engine. Tags are no longer static labels. They live in an XY Tag Field, can be moved to reshape influence, and can be “heated” to pull more images into their orbit. Under these tags, Atlas forms multi-column stalactites at intersections, producing structured gradients that are navigable by depth.

New: XY Tag Field (move tags) New: Tag Heat Controls New: Multi-column intersections New: Depth Carousel + Axis Scroll New: Column comparisons (side-by-side)

How multi-column stalactites form

Think of each tag as a “gravitational body” placed in a 2D field. When tags are close, they create strong combined influence. The system generates columns under these influence points. When multiple tag influences intersect, you get multiple columns (stalactites) that reflect the strongest match to the primary tag and secondary alignment to nearby tags.

Example: Tag placement changes output

If texture sits between size and opacity, columns generated under texture will preferentially align with both size and opacity (while still maximizing texture as the primary target).

Move texture farther away and the intersection weakens, producing different columns.
Heat meaning (non-invasive, intuitive):

Default heat values produce the “truest” placement relative to the field. Increasing heat on a tag pulls more images toward that tag’s region, even away from competing tags.

In UI terms: heat increases “mass” and enlarges that tag’s capture radius.
Outcome: no tag is inert. Every tag affects the formation of columns and intersections through position and heat.

Navigation: Depth Carousel + Axis Scrolling

How users explore the structure

Depth Carousel (scroll through layers)

Each column is a ranked gradient from closest to furthest relative to an anchor (image, tag, or multi-tag intersection). Users traverse this depth using a carousel dial:

  • Large dial: move layer-by-layer through the entire column or cluster depth
  • Small dial: when a column is selected, move image-by-image within that single column
  • Numeric input: jump directly to a depth index (example: 172 of 1,240)

Axis Scrolling (see structure from multiple viewpoints)

Atlas provides multiple viewpoints to make complex clusters understandable:

  • Z-dimension layer overview: a top-down view of the “surface” layer, then step down by depth
  • X-dimension layer overview: a side view to understand column height and density
  • Y-dimension layer overview: another side perspective, useful for comparing adjacent columns
  • Column-by-column comparison: pin columns side-by-side to compare gradients directly
Practical meaning:

Z overview helps you understand the “top layer” of strongest matches. Side views help you see how deep and dense a column is. Column comparison helps you decide which branch truly fits a concept better.

Non-invasive Creativity Nudges

Creative guidance without surveillance

Tag Saturation Prompt

To encourage variety and precision without policing creativity, Atlas offers gentle prompts based on a user’s tag history:

“You’ve used this tag 500 times. Continue, or add a modifier?”

If user adds a meaningful modifier:
“You’ve made a new tag. Congratulations.”

Sensical Tag Check (optional)

Atlas can optionally flag nonsensical or non-differentiating tags, while still allowing overrides:

“That’s not sensical.”
“This tag overlaps 92% with an existing tag. Refine?”
“Proceed anyway?”
Design principle: nudges are optional, transparent, and never invasive. Users remain in control.

Data Model: What Each Image Carries

To keep the system inspectable

Per-image metadata (core)

  • Tag-field coordinate: where it sits relative to the XY tag field
  • Unique Complex Marker: compact identifier describing the column configuration
  • Descriptor: explains relationship to all tags, not just the nearest two
  • Context snapshot: which tags were active, their XY positions, and their heats

Per-column metadata (core)

  • Primary driver: which tag or image anchor created the column
  • Neighbor tag layout: which tags were nearby and how close they were
  • Heat map: tag heats used to generate this structure
  • Depth statistics: length, density zones, and discontinuity points
Why this matters: Without metadata, a spatial system feels like a magic trick. With metadata, Atlas becomes a readable instrument.

Machine Learning: Gradient Columns as Training Data

How Atlas becomes smarter

Anchor Pair → Model Gradient → Curated Column

Atlas can train models using a consistent artifact: the Gradient Column. Human curators provide two endpoints:

  • Top Anchor: the closest image to the target concept
  • Bottom Anchor: the furthest image still meaningfully connected

The model generates 10 to 20 images between the anchors, then users curate the best ordering. The result is a structured, distance-aware dataset that teaches models how “meaning fades” over depth.

Training outcome: models learn not just what a concept looks like, but what it looks like at different distances.

Why this is better than basic tagging

  • Tagging labels identity. Gradient columns label distance and decay.
  • Columns create stable supervision for ranking, recommendation, and generation.
  • User curation provides consensus structure, not single-person noise.

Conclusion

What someone should remember

Atlas evolves from a powerful similarity browser into a full spatial cognition engine:

  • v1.0: Image similarity mapping and hub-based navigation
  • v2.0: Sphere-based world navigation and slice exploration
  • v3.0: Atlas Dimensions: XY tag field, heat controls, multi-column stalactites, and depth + axis traversal
Bottom line: Atlas turns archives into fields and forces. It makes meaning navigable.
